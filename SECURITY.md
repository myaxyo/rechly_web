# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Until a dedicated security contact is published, report vulnerabilities privately to the project maintainer through the contact address configured for the deployment or repository.

Include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Suggested mitigations, if known

## Response Expectations

The goal is to acknowledge valid reports promptly, confirm impact, and ship a fix before public disclosure when feasible.

## Scope

This policy covers the web app, the ML service, deployment configuration committed to the repository, and any supported release artifacts.
